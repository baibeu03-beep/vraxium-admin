## 3. 분류

### SoT 일치 항목
- 주차 카드(weekly-cards) 전 경로: front 프록시 = admin HTTP = direct snapshot (실증 0 diff)
- check_threshold / checks_migrated / point.check (게이트 v18 전 경로 단일 수렴)
- 실무 경험 평점 (단일 테이블·단일 임계상수, front는 proxy 표시만)
- seasonPointSummary (front=admin 실증 일치)
- demoUserId vs 일반 경로 (동일 로더, 운영 게이트 정상)
- 정본 성공주차 카운트 3경로 (stats-cards / growth_stats / confirmed-success-weeks)

### SoT 불일치 항목
- **front weekly-growth 독립 구현** (A — 실증: 27vs26, 1vs0, 42vs26, W13 성공vs집계중)
- **front 카드상세 휴식 판정 = legacy 컬럼 직독** (B — 정의 충돌 지속)
- **전환주 휴식 정의 상충** (C/T5 — resolver true vs front·컬럼 false, 실데이터 2건)
- front /api/profile/summary (D — 전환·공표 미적용 + 레거시 시즌 체계)
- front Cluster4Content 클라이언트 누적 계산 (E — 게이팅 입력 오염)

### 즉시 수정 필요 항목
1. **(E) 라인 주차범위 게이팅 입력 교정** — 클라이언트 raw 계산 대신 snapshot `accumulatedApprovedWeeks`(또는 admin acc) 직독. 미공표/전환 성공 +1로 minWeek 경계에서 라인이 일찍/잘못 열릴 수 있음 (실증 27 vs 26).
2. **(A) front weekly-growth publish 게이트** — 최소한 미공표 주차를 "성공"으로 반환하지 않도록 result_published_at 게이트 추가(공개 엔드포인트라 stale 사본·외부 조회 모두 확정 전 결과를 봄).
3. **(D) profile/summary 전환주 제외 + 공표 게이트** — 사이드바 시즌 이력 수치가 정본과 1+ 어긋남.

### 마이그레이션 전에 반드시 막아야 할 항목
1. **전환주 is_official_rest 방침 확정 (T5 2건)** — legacy 컬럼 동기화 마이그레이션에서 resolver 값을 그대로 복사하면 전환주 2건이 true가 되어 **front 카드상세가 전환주를 공식휴식으로 표시하는 역회귀** 발생. 동기화 스크립트는 전환주를 제외(또는 front 정의 기준)해야 함. ← winter W5/W8 수정과 같은 스크립트 계열을 재사용할 경우 특히 주의.
2. **front weekly-growth 독립 구현 방치 금지** — 레거시 통합 라인(v17)·check 게이트(v18) 이관이 진행될수록 front raw uws 카운트와 정본 판정의 격차가 커짐. 이관 전에 front weeklyCards/growthStats를 admin proxy로 전환하거나 폐기 결정 필요.
3. **check-threshold 참여자 조회 1000행 cap** — `adminWeekRecognitionsData.ts:724-727` uws 참여자 select가 order+range 페이지네이션 없음(PostgREST 1000행 cap 메모리 항목과 동일 패턴). 현재 인원(≈120)은 안전하나 대량 이관 후 주차당 1000명 초과 시 일부 snapshot 미재계산.
4. **원장 직접 수정 시 snapshot 무효화 누락** — 마이그레이션 스크립트가 uws/points를 직접 쓰면 자동 무효화 안 됨 → `recalcUserGrowthStats` + `backfill:weekly-card-snapshots` 필수(기존 정책 재확인).
5. **sponsor-meta / career-evaluations 는 stale 마킹만** — 대량 수정 후 즉시 재계산 안 됨. 마이그레이션 후속으로 일괄 재계산 배치 필요.

## 4. 부수 관찰 (비차단)

- `[weekly-growth][area-8] user_team_parts / user_role_history` 테이블 부재 경고 — 알려진 fallback 동작(메모리 기존 항목).
